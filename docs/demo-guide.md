# Hello AI 无障碍辅助 - 演示指南

## 一、演示准备

### 1.1 环境要求

| 项目 | 要求 |
|------|------|
| **浏览器** | Chrome 90+ / Edge 90+ / Safari 15+ / Firefox 90+ |
| **设备** | 电脑或手机（推荐手机体验更佳） |
| **网络** | 首次访问需要网络，之后可离线使用 |
| **权限** | 需要允许摄像头和麦克风权限 |

### 1.2 演示前准备

1. **打开项目**
   - 在线访问：https://hello-ai.netlify.app/
   - 本地运行：下载项目后执行 `python -m http.server 8080`，浏览器打开 `http://localhost:8080`

2. **检查权限**
   - 确保浏览器允许摄像头权限
   - 确保浏览器允许麦克风权限
   - 确保浏览器允许语音合成权限

3. **测试设备**
   - 电脑：确保摄像头和麦克风正常工作
   - 手机：确保前后摄像头可切换

## 五、常见问题解答

### 5.1 技术问题

**Q: 为什么选择纯前端技术？**
A: 纯前端技术可以零成本部署，无需后端服务器，降低了使用门槛。同时，PWA 技术让应用离线可用，网络差也能用。

**Q: 手势识别的准确率如何？**
A: 对于预置的 32 种手势，准确率在 90% 以上。对于自定义手语，需要通过训练系统采集样本，准确率取决于训练数据的质量和数量。

**Q: 如何保证用户隐私？**
A: 所有数据都在本地处理，不上传服务器。摄像头画面、语音数据、训练数据都存储在用户设备上。

### 5.2 使用问题

**Q: 支持哪些浏览器？**
A: 支持 Chrome 90+、Edge 90+、Safari 15+、Firefox 90+。推荐使用 Chrome 或 Edge 获得最佳体验。

**Q: 手机端如何使用？**
A: 手机端可以直接访问项目网址，或者安装 PWA 到桌面。手机端会自动适配响应式布局，并使用后置摄像头（视障模式）。

**Q: 如何导入 vivo 手语训练数据？**
A: 在听障辅助模式中，点击"导入vivo数据"按钮，选择 vivo 格式的 JSON 文件即可。系统会自动检测数据格式并导入。

### 5.3 未来计划

**Q: 未来有什么计划？**
A: 短期计划：邀请残障朋友试用，收集真实反馈；接入 vivo 手语训练数据，提升识别准确率。中期计划：与残障公益组织合作，推动实际落地。长期愿景：接入大语言模型，提升场景理解能力。

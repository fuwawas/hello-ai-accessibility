# 【Hello AI 科技致善】用 AI 打破无障碍壁垒

## ① 摘要

Hello AI 是一款面向听障、视障、认知障碍、肢体障碍及老年群体的无障碍辅助 PWA 应用。通过 MediaPipe 手势识别将手语实时转为语音播报，通过摄像头为视障用户提供环境感知，通过语音交互解放肢体障碍用户的双手。纯浏览器端运行，零服务器成本，首次访问后可离线使用。

## ② 真实场景与需求

### 目标人群
- **听障人士**（中国约 2780 万）：无法通过语音表达需求，与健听人沟通困难
- **视障人士**（中国约 852 万）：无法获取视觉信息，出行和生活依赖他人
- **认知障碍人士**：难以独立完成日常任务（吃药、散步等）
- **肢体障碍人士**：无法使用触屏操作手机
- **老年人**（中国约 2.97 亿）：不熟悉智能设备操作

### 痛点描述
- 听障人士去医院看病，无法与医生沟通症状
- 视障老人独自在家，无法分辨药品、感知环境
- 肢体障碍用户无法用双手操作手机

### 现有方案的不足
- 专业手语翻译 APP 需要联网、收费、覆盖词汇少
- 读屏软件只能读文字，无法描述摄像头画面
- 语音助手（Siri/小爱）不支持无障碍场景定制

## ③ 作品介绍

Hello AI 是一个纯前端 PWA 应用，包含 5 大辅助模式：

| 模式 | 核心功能 | 技术 |
|------|---------|------|
| 🗣️ 听障辅助 | 手势识别→语音播报、手语训练、自定义词汇 | MediaPipe + KNN + Web Speech |
| 👁️ 视障辅助 | 实时感知、场景描述、颜色/光线检测、障碍提醒、紧急求助 | 摄像头 + 语音合成 |
| 🧠 认知辅助 | 6 种日常任务步骤引导（吃药、散步等） | 任务模板 + 语音 |
| 🦿 肢体辅助 | 语音控制、命令识别 | Web Speech API |
| 👴 老年辅助 | 大按钮、语音交互、8 个快捷操作 | 简化 UI + 语音 |

### 技术亮点
- **纯浏览器端**：无需后端服务器，零部署成本
- **PWA 离线**：首次访问后可离线使用
- **动态手势识别**：追踪 15 帧手势轨迹，识别 32 种预置手势
- **KNN 自定义训练**：用户可训练自定义手势词汇
- **多策略缓存**：Service Worker 实现三级缓存策略

## ④ 用 SOLO 实现的过程

### 任务拆解
1. 需求分析 → 确定目标用户和核心功能
2. 技术选型 → MediaPipe + Web Speech API + PWA
3. UI 设计 → 莱卡暗金色调，温暖不冰冷
4. 核心开发 → 5 大辅助模式逐一实现
5. 手势识别 → 静态 KNN + 动态序列识别
6. 代码重构 → 单文件拆分为 12 个模块
7. PWA 优化 → Service Worker + 骨架屏 + 安装引导
8. 落地页 → 充满人情味的产品展示页
9. 无障碍 → WCAG AA 标准适配
10. 测试验收 → 多轮功能检查和修复

### 用到的 SOLO 能力
- **代码编写**：3000+ 行 JavaScript，12 个模块
- **代码调试**：修复 10+ 个高严重度 bug（内存泄漏、XSS、兼容性等）
- **代码重构**：app.js 拆分为模块化架构
- **Web 搜索**：调研手语数据集、PWA 最佳实践
- **文件生成**：SVG 图标、README、LICENSE、提交帖

### 踩过的坑
1. **KNN 阈值问题**：距离阈值 2.0 太严格，导致所有结果被过滤；后改为置信度阈值
2. **双重 KNN 覆盖**：两个地方同时做 KNN 分类，互相覆盖结果
3. **IndexedDB 残留**：清除训练数据只清内存没清 IndexedDB，旧数据一直残留
4. **标准手势误识别**：MediaPipe 把各种手势误识别为 Open_Palm
5. **ES Module 作用域**：拆分模块后 $ 函数在模块中不可用
6. **Object.assign 时机**：class 声明前访问原型会触发 TDZ 错误

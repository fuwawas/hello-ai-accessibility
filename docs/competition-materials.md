## 一、项目概述

### 1.1 项目名称

### 1.2 项目定位

## 三、需求分析

### 3.1 目标人群

| 人群 | 中国数量 | 核心痛点 |
|------|----------|----------|
| **听障人士** | 约 2,780 万 | 手语是母语，但健听人看不懂，日常沟通困难 |
| **视障人士** | 约 852 万 | 无法获取视觉信息，出行和生活依赖他人 |
| **认知障碍人士** | 数百万 | 难以独立完成日常任务（吃药、散步等） |
| **肢体障碍人士** | 数百万 | 无法使用触屏操作手机 |
| **老年人** | 约 2.97 亿 | 不熟悉智能设备操作，紧急情况无法快速求助 |

### 3.2 现有方案不足

| 方案 | 问题 |
|------|------|
| 专业辅助设备 | 价格高昂（数千元），普通家庭难以承受 |

## 四、作品介绍

### 4.1 五合一辅助模式

#### 🗣️ 听障辅助模式
- **实时手势识别转语音文字**
- **动态手势序列识别**：追踪 15 帧手势轨迹，识别 32 种预置手势
- **KNN 自定义训练**：用户可训练自定义手语词汇
- **手语输入模式**：手语组合成句子并朗读
- **训练数据管理**：支持导入/导出/清除训练数据
- **vivo 数据接口**：预留 vivo 手语训练数据导入接口

#### 👁️ 视障辅助模式
- **实时环境感知**：亮度、颜色、场景判断
- **障碍物检测**：帧差法检测障碍物并语音提醒
- **屏幕朗读**：朗读当前页面文字内容
- **颜色识别**：识别摄像头前方的主要颜色
- **光线检测**：持续监测环境光线亮度变化
- **紧急求助**：GPS 定位 + 联系人管理 + 短信发送
- **手势快捷操作**：7 种手势控制（大拇指=确认，握拳=返回等）

#### 🧠 认知障碍辅助模式
- **6 种内置任务模板**：打电话、吃药提醒、查看天气、散步、吃饭、紧急求助
- **自定义工作流程**：家人可为患者创建步骤引导
- **语音引导**：每步语音引导 + 选项按钮
- **大字体高对比度**：自动启用无障碍模式

#### 🦿 肢体障碍辅助模式
- **语音命令识别**：连续监听模式
- **8 种语音命令**：返回、主页、朗读、大字体、高对比度、帮助、停止、紧急求助
- **语音波形动画**：实时反馈语音状态
- **回声防止**：播报期间自动暂停识别

#### 👴 老年人辅助模式
- **问候语**：根据时间段自动切换早/午/晚
- **8 个大按钮快捷操作**：紧急求助、联系家人、吃药提醒、天气、时间、新闻、音乐、帮助
- **大字体模式**：自动启用大字体
- **震动反馈**：紧急求助时震动提醒

### 4.2 通用功能

#### 声音修复模式
- 语音识别 + 清晰声音输出
- 手动文字输入朗读
- 14 个常用短语快捷按钮
- 语速/音调可调
- 历史记录回放

#### 无障碍设置
- 高对比度模式切换
- 大字体模式切换
- 语音播报开关
- 语速调节（0.5x - 2.0x）
- 音调调节（0.5x - 2.0x）

## 五、技术实现

### 5.1 技术栈

| 层级 | 技术 |
|------|------|
| **标记语言** | HTML5（语义化标签、ARIA 无障碍属性） |
| **样式** | 纯 CSS3（CSS 自定义属性/设计令牌体系） |
| **脚本** | 原生 JavaScript ES6+（ES Modules） |
| **手势识别** | MediaPipe Tasks Vision（WASM + Gesture Recognizer） |
| **语音合成** | Web Speech API (SpeechSynthesis) |
| **语音识别** | Web Speech API (SpeechRecognition) |
| **手语分类** | 自研 KNN 分类器 + 动态手势序列识别器 |
| **数据存储** | localStorage + IndexedDB |
| **离线支持** | Service Worker（三级缓存策略） |
| **PWA** | manifest.json + Service Worker |
| **部署** | Netlify（静态站点托管） |

### 5.3 技术亮点

#### 1. 纯浏览器端 AI
- MediaPipe WASM 在浏览器中运行手势识别，无需后端
- 零服务器成本，零部署成本

#### 2. 零依赖架构
- 不使用 React、Vue、jQuery 等任何第三方框架
- 纯原生 HTML/CSS/JS，加载极快

#### 3. PWA 离线可用
- Service Worker 三级缓存策略
- 首次访问后可离线使用

#### 4. 动态手势识别
- 追踪 15 帧手势轨迹
- 32 种预置动态手势模板
- 自研 KNN 分类器支持自定义训练

#### 5. 无障碍设计
- CSS 变量体系支持主题切换
- 跳过导航链接、ARIA 标签全覆盖
- 键盘完全可操作
- `prefers-reduced-motion` 媒体查询
- 最小触摸区域 48px（WCAG 标准）

#### 6. 响应式设计
- 三断点（1024px 平板 / 640px 手机 / 380px 超小屏）
- 手机端深度适配：安全区域、触摸优化、摄像头切换

## 六、开发过程

### 6.1 任务拆解

**第一阶段：基础架构搭建**
- 用 SOLO 生成项目骨架：HTML 结构、CSS 设计系统、JS 模块架构
- 设计了莱卡黑白摄影 + 暖金色点缀的视觉风格
- 建立了 CSS 变量体系（设计令牌），支持主题切换

**第二阶段：五个辅助模式开发**
- 逐个模式开发：先写 HTML 模板，再实现 JS 逻辑
- 每个模式独立模块化，互不干扰
- SOLO 帮助快速生成了大量的 DOM 操作代码和事件处理逻辑

**第三阶段：AI 能力集成**
- 集成 MediaPipe Tasks Vision 实现手势识别
- 实现 KNN 分类器支持自定义手语训练
- 用 Web Speech API 实现语音识别和语音合成

**第四阶段：优化与打磨**
- 手机端适配：安全区域、触摸优化、摄像头切换
- PWA 支持：Service Worker、离线缓存、安装提示
- 无障碍增强：ARIA 标签、键盘操作、焦点管理

### 6.2 用到的 SOLO 能力

| 能力 | 应用 |
|------|------|
| **代码编写** | 3000+ 行 JavaScript，12 个模块 |
| **代码调试** | 修复 10+ 个高严重度 bug（内存泄漏、XSS、兼容性等） |
| **代码重构** | app.js 拆分为模块化架构 |
| **Web 搜索** | 调研手语数据集、PWA 最佳实践 |
| **文件生成** | SVG 图标、README、LICENSE、提交帖 |

### 6.3 踩过的坑

1. **MediaPipe WASM 加载**：CDN 在国内不稳定，改为本地文件 + CDN 回退方案
2. **手机端语音解锁**：iOS/Android 要求用户交互后才能播放语音，加入了点击解锁机制
3. **手势识别性能**：手机端 GPU 不稳定，自动降级到 CPU 模式
4. **手语训练数据**：IndexedDB 版本升级时旧数据残留，加入版本号自动清除机制
5. **KNN 阈值问题**：距离阈值 2.0 太严格，导致所有结果被过滤；后改为置信度阈值
6. **双重 KNN 覆盖**：两个地方同时做 KNN 分类，互相覆盖结果
7. **标准手势误识别**：MediaPipe 把各种手势误识别为 Open_Palm
8. **ES Module 作用域**：拆分模块后 $ 函数在模块中不可用
